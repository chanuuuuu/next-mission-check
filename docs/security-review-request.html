선교 대원 조회 시스템 (Inquery) 보안 검토 요청

작성일 2026-06-02 대상 기능 /inquery 경로 (대원 조회 + 관리자) 플랫폼 Next.js 16 / Neon DB (PostgreSQL) / Vercel Edge
본 문서는 선교 등록 대원 조회 시스템(Inquery)에 대한 보안 검토를 요청하기 위해 작성되었습니다.
수집하는 개인정보는 이름과 전화번호 끝 4자리에 한정됩니다. 전체 전화번호는 서버에 도달하기 전에 Google Apps Script 단에서 잘리며, 서버는 끝 4자리만 수신합니다.
모집 기간이 종료되는 즉시 DB 전체를 삭제할 예정입니다.
검토 범위 — 이 문서는 /api/inquery/* 엔드포인트 및 mission_registrations 테이블에 한정합니다. 체크인·QR·SSE 기능은 별도 시스템으로 이 요청 범위에 포함되지 않습니다.
01 데이터베이스 스키마 — mission_registrations

Inquery 기능이 사용하는 테이블은 mission_registrations 단 하나입니다. 아래는 전체 컬럼 명세입니다.

컬럼명 타입 제약 설명 개인정보
id UUID PK, gen_random_uuid() 레코드 고유 식별자 없음
department_main VARCHAR NOT NULL 대분류 부서 (2청 / 기타부서 / 청장년) 없음
sub_department_1 VARCHAR NOT NULL 세부 부서 1계층 (진, 소속부서) 없음
sub_department_2 VARCHAR NULL 세부 부서 2계층 (팀 / 연계교회명 / 기타부서: NULL) 없음
small_group VARCHAR NULL 소그룹 이름 (청장년의 '소속 목장' 전용) 없음
name VARCHAR NOT NULL 이름 이름
phone_last_four VARCHAR(4) NOT NULL 휴대폰 번호 뒤 4자리만 저장 전화번호 끝 4자리
church_name VARCHAR NULL 연계교회 이름 (2청 전용) 없음
arrival_time VARCHAR NULL 연계교회 도착 예상 시간 없음
use_personal_car BOOLEAN NULL 자차 이용 여부 없음
use_car_during_mission VARCHAR NULL 선교 기간 중 자차 이용 여부 (텍스트 응답) 없음
use_return_bus BOOLEAN NULL 복귀 버스 탑승 여부 없음
schedule_survey VARCHAR NULL 참여 일정 및 이동수단 조사 응답 (텍스트) 없음
payment_status BOOLEAN NOT NULL, Default FALSE 등록비 납부 여부 (관리자 전용 토글) 없음
created_at TIMESTAMPTZ NOT NULL, Default NOW() 최초 등록 일시 없음
updated_at TIMESTAMPTZ NOT NULL, Default NOW() 최근 수정 일시 없음
개인정보 최소화 설계 포인트
  • ✅ 수집하지 않는 정보: 전체 전화번호, 이메일, 주소, 생년월일, 계좌번호 — 해당 컬럼 자체가 스키마에 존재하지 않음
  • ✅ 전화번호는 Google Form 제출 즉시 GAS 단에서 .slice(-4) 처리 — 서버에 원본이 도달하지 않음
  • ✅ payment_status는 Boolean만 저장 — 결제 수단·금액·카드 정보 없음
유니크 제약 및 중복 처리
-- 복합 유니크 키 (PostgreSQL 15+ NULLS NOT DISTINCT)
CONSTRAINT uq_registration UNIQUE NULLS NOT DISTINCT (
  sub_department_1, sub_department_2, name, phone_last_four
)

-- UPSERT 시 payment_status 는 의도적으로 제외
-- → 관리자가 수동 체크한 납부 상태가 재제출로 초기화되지 않도록 격리
02 웹훅 구조 및 수집 정보

Inquery 데이터 유입 경로는 수신 웹훅 1개와 관리자 수동 동기화 1개입니다. 발신 웹훅은 Discord 오류 알림 전용으로, 정상 등록 데이터는 Discord로 전송되지 않습니다.

항목내용
인증 요청 헤더 x-webhook-secret → 서버 환경변수 WEBHOOK_SECRET 일치 여부 확인. 불일치 시 401 즉시 반환, 이후 처리 없음
수신 페이로드 { department: string, namedValues: Record<string, string[]> } — Google Forms namedValues 구조 그대로 수신
전화번호 처리 서버 수신 시 String(phone).slice(-4) → 끝 4자리만 DB 저장, 원본 즉시 폐기
부서 검증 FORM_MAPPINGS에 등록된 부서명만 처리. 알 수 없는 부서명은 Discord 알림 후 400 반환
필수 필드 누락 sub_department_1, name, phone_last_four 중 하나라도 비면 Discord 알림 후 400 반환

전화번호 처리 웹훅 동일 — .slice(-4) 처리 후 저장

알림 종류발송 조건포함 정보
동기화 실패 웹훅/수동 동기화 중 필수 필드 누락 또는 알 수 없는 부서명 부서명, 이름(실패 케이스만), 오류 원인, 발생 시각
  • ✅ Discord Webhook URL은 환경변수 DISCORD_WEBHOOK_URL — 코드 내 하드코딩 없음
  • ✅ URL 미설정 시 알림 전송을 조용히 건너뜀. Webhook 발신 실패가 메인 흐름을 차단하지 않음
  • ✅ 정상 등록·체크인 데이터는 Discord로 전송되지 않음 — 오류 상황에서만 발신
03 Inquery API 목록 및 보안 조치

모든 /api/inquery/* 엔드포인트는 Vercel Edge Runtime에서 실행됩니다.

엔드포인트 메서드 접근 주체 인증·보안 조치
/api/inquery/auth POST 관리자 bcrypt 해시 비교 (ADMIN_PIN_HASH env). HttpOnly 쿠키 발급, secure:true(prod), 8시간 만료
/api/inquery/auth DELETE 관리자 쿠키 maxAge: 0으로 즉시 만료 처리
/api/inquery/departments GET 일반 이용자 인증 불필요. department_hierarchy 테이블에서 부서 트리만 반환 (개인정보 없음)
/api/inquery/registrations/departments GET 일반 이용자 Cascading dropdown용 세부 부서 목록 반환. 개인정보 컬럼 미포함
/api/inquery/registrations GET 일반 이용자 / 관리자 비인증: 결과 최대 30건으로 제한 (rows.slice(0, 30)). 관리자 쿠키 확인 시 전체 반환
/api/inquery/registrations POST 관리자 관리자 UI 전용 수동 대원 추가
/api/inquery/registrations/[id]/payment PATCH 관리자 inquery_admin_session 쿠키 authenticated 검증 필수. 미인증 시 401
/api/inquery/sync/webhook POST GAS (서버 간) x-webhook-secret 헤더 검증. 불일치 시 401 즉시 반환
/api/inquery/sync/manual POST 관리자 관리자 쿠키 검증 필수. Google Sheets API는 Service Account로만 접근
공통 보안 사항
  • ✅ 모든 DB 쿼리는 파라미터화된 템플릿 리터럴 (neon() HTTP 드라이버) 사용 — SQL 인젝션 방지
  • ✅ 관리자 PIN은 평문 비교 없이 bcrypt 해시(ADMIN_PIN_HASH)로만 검증
  • ✅ 인증 쿠키: httpOnly: true, sameSite: 'lax', 프로덕션 secure: true, 8시간 만료
  • ✅ 모든 시크릿은 Vercel 환경변수에만 저장 — 저장소 코드에 미포함
사용 환경변수 목록
변수명용도코드 내 하드코딩
DATABASE_URLNeon DB 연결 문자열없음
WEBHOOK_SECRETGAS Webhook 인증 토큰없음
ADMIN_PIN_HASH관리자 PIN bcrypt 해시없음
DISCORD_WEBHOOK_URL오류 알림 Discord URL (선택)없음
GOOGLE_CLIENT_EMAILSheets API Service Account 이메일없음
GOOGLE_PRIVATE_KEYSheets API Service Account 키없음
GOOGLE_SHEET_ID_2CHUNG2청 부서 Google Spreadsheet ID없음
GOOGLE_SHEET_ID_ETC기타부서 Google Spreadsheet ID없음
GOOGLE_SHEET_ID_ADULT청장년 Google Spreadsheet ID없음
04 데이터 삭제 및 운용 종료 계획

본 시스템은 선교 모집 기간에만 한시 운용됩니다. 모집 기간이 종료되는 즉시 아래 순서로 데이터와 접근 권한을 제거합니다.

단계작업비고
① mission_registrations 테이블 DROP 이름·전화번호 끝 4자리 포함 — 최우선 삭제
② Neon DB 프로젝트 전체 삭제 또는 나머지 테이블 DROP 관련 테이블 일괄 제거
③ Vercel 환경변수 전체 삭제 또는 프로젝트 비활성화 DATABASE_URL, WEBHOOK_SECRET, GOOGLE_PRIVATE_KEY 등
④ Google Service Account 키 폐기 Google Cloud Console에서 키 삭제
⑤ Discord Webhook URL 무효화 Discord 채널에서 Webhook 삭제
  • 📋 삭제 완료 후 Neon 쿼리 이력 또는 삭제 확인 화면을 보안담당자에게 공유 가능합니다
  • 📋 Google Sheets 내 원본 Form 응답 데이터의 삭제는 각 부서 담당자에게 별도 안내 예정입니다
